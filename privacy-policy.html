DataPulse Privacy Policy

Last updated: January 19, 2026

DataPulse is a private desktop application used exclusively by its operator for internal business analysis. We are committed to protecting data privacy and maintaining compliance with applicable regulations.

Application Purpose

DataPulse is a Windows desktop application used for private business intelligence and market research purposes. The application operates as an internal tool and is not distributed to third parties.

Data Collection

DataPulse processes business-related data necessary for its analytical functions. This includes product information, market data, and user preferences required for application operation.

Data Storage

All data processed by DataPulse is stored locally on the user's device. The application does not transmit data to external servers or cloud services except when explicitly using third-party API integrations (see below). Industry-standard AES-256 encryption is used to protect sensitive information including API credentials.

Data Sharing

We do not share, sell, rent, or distribute any data to third parties. All information processed by the application remains strictly private and is used solely for internal business purposes by the application operator.

Third-Party API Services

DataPulse integrates with authorised third-party APIs to provide market intelligence features. Each integration operates under strict compliance with the respective service provider's terms of use and data protection requirements.

HMRC VAT API

DataPulse connects to HMRC's VAT API to verify UK VAT registration numbers for supplier credibility assessment. When you use the VAT validation feature:

  • The VAT number you enter is sent to HMRC for verification
  • HMRC returns the registered business name and validation status
  • We cache results locally for 24 hours to reduce API calls
  • We do not store HMRC data permanently or share it with third parties
  • You can delete cached VAT data at any time through Settings
  • Validation is triggered manually only - never automated

For HMRC's privacy policy, see: HMRC Developer Hub Privacy

Google Custom Search API

DataPulse optionally uses Google's Custom Search API to discover company websites for supplier research. When you use the website discovery feature:

  • The company name is sent to Google for searching
  • Google returns search results which may include URLs
  • We cache discovered URLs locally to avoid repeated searches
  • We do not store search query history
  • Searches are triggered manually only - never automated

Google's search is subject to their Privacy Policy. You can disable Google API integration in Settings â†’ API Keys.

Companies House API

DataPulse connects to the UK Companies House API to retrieve publicly available company information for supplier discovery. When you search for companies:

  • Search queries and company numbers are sent to Companies House
  • Public company data (name, address, directors, filings) is retrieved
  • Results are cached locally for 24 hours to reduce API calls
  • All data is publicly available under the Open Government Licence v3.0

For more information, see: Companies House API Documentation

Amazon Selling Partner API

DataPulse optionally connects to Amazon's Selling Partner API to retrieve product data and pricing information. When enabled:

  • Product identifiers (ASINs) are sent to Amazon for lookup
  • Amazon returns current pricing, offers, and product details
  • Your Amazon seller credentials are stored locally with AES-256 encryption
  • We do not share your Amazon data with third parties
  • API usage is logged locally for audit compliance

Amazon SP-API usage is subject to Amazon's Developer Policy.

Keepa API

DataPulse optionally integrates with Keepa for historical Amazon sales data. When enabled:

  • Product identifiers are sent to Keepa for historical data lookup
  • Keepa returns price history, sales rank trends, and availability data
  • Your Keepa API key is stored locally with AES-256 encryption
  • Results are cached to reduce API token usage

Data Retention

Data is retained locally for as long as necessary for business purposes. API response caches expire after 24 hours. Users may clear stored data at any time through the application settings. Uninstalling the application removes all associated data.

Security

We implement appropriate technical measures to protect data against unauthorised access, alteration, or destruction. This includes:

  • AES-256 encryption for all stored API credentials
  • Windows DPAPI for additional credential protection
  • OAuth 2.0 with secure token refresh for Amazon SP-API
  • Audit logging of all API interactions
  • No credentials hardcoded or embedded in application code

User Rights

Users maintain full control over their data and may:

  • Access and review stored information
  • Delete cached data at any time
  • Export their data in standard formats
  • Remove all data by uninstalling the application
  • Disable any third-party API integration
  • Clear individual API caches through Settings

Compliance

This application and its data handling practices comply with UK data protection regulations and the terms of service of any third-party APIs utilised.

Policy Updates

This privacy policy may be updated periodically. The "Last updated" date indicates the most recent revision. Continued use of the application constitutes acceptance of the current policy.

Contact

For privacy-related inquiries regarding this policy, please contact DataPulse Analytics through the application's help system.
